```
┌──────────────────────────────────────────────────────────────┐
│                      사용자 진입점                             │

                              ↓
                    ┌─────────────────┐
                    │  로그인 필요?    │
                    └─────────────────┘
                         │           │
                      Yes│           │No
                         ↓           ↓
              ┌──────────────┐   ┌──────────────┐
              │ 로그인 페이지 │   │  대시보드     │
              └──────────────┘   └──────────────┘
                     ↓                   ↓

              └──────────────┘   └──────────────┘

   └────────┘   └────────┘       └────────┘   └────────┘

## 주요 사용자 시나리오

### 시나리오 1: 정책 담당자의 일일 모니터링

**목표**: 오늘 새로 올라온 공지사항 확인

```
[시작] 사용자가 웹사이트 접속
   ↓
[대시보드 로딩]
   - 통계 카드 표시 (총 게시물, 오늘 수집 등)
   - "오늘 수집: 324건" 확인
   ↓
[최근 게시물 스캔]
   - 오늘 날짜 게시물에 집중
   - 제목만 빠르게 훑어봄
   ↓
[관심 게시물 발견]
   - "친환경 정책 발표" 제목 클릭
   ↓
[원문 확인]
   - 새 탭에서 원문 페이지 열림
   - 내용 확인
   ↓
┌─────────────────┐
│ 중요한가?        │
└─────────────────┘
   │           │
 Yes│           │No
   ↓           ↓
[북마크 추가]  [다음 게시물로]
   ↓           ↓
[알림 표시]    [계속 스캔]
   ↓           ↓
[완료]        [루프]
```

**소요 시간**: 3-5분
**주요 액션**: 스캔(10건), 클릭(3건), 북마크(1건)

---

### 시나리오 2: 입찰 담당자의 정밀 검색

**목표**: 부산 지역 청사 보수 관련 입찰 공고 찾기

```
[시작] 전체 게시물 메뉴 클릭
   ↓
[필터 설정]
   ├─ 게시판: "입찰공고" 선택
   ├─ 기관: "부산청" 선택
   └─ 날짜: "최근 30일" 선택
   ↓
[필터 적용]
   - 로딩 인디케이터 표시
   - 결과: 47건
   ↓
[검색어 입력]
   - "청사 보수" 입력
   - 실시간 필터링 (디바운스 300ms)
   ↓
[결과 정제]
   - 검색 결과: 3건
   - 검색어 하이라이트 표시
   ↓
[정렬 조정]
   - "작성일" 클릭 (최신순)
   ↓
[게시물 검토]
   ┌─────────────────┐
   │ 각 게시물 확인   │
   └─────────────────┘
        ↓      ↓      ↓
     게시물1  게시물2  게시물3
        ↓      ↓      ↓
   [북마크]  [원문]  [넘어감]
        ↓      ↓
   [메모 추가] [다운로드]
        ↓      ↓
      [완료]  [완료]
```

**소요 시간**: 5-10분
**주요 액션**: 필터(3개), 검색(1회), 정렬(1회), 북마크(1건), 원문 확인(2건)

---

### 시나리오 3: 언론인의 모바일 빠른 확인

**목표**: 이동 중 스마트폰으로 오늘 인사발령 확인

```
[시작] 모바일 브라우저에서 접속
   ↓
[모바일 레이아웃 로딩]
   - 통계 카드 2열 표시
   - 차트 접힌 상태
   ↓
[햄버거 메뉴 터치]
   - 좌측에서 네비게이션 슬라이드 인
   ↓
[빠른 필터 사용]
   - "인사발령" 체크박스 선택
   - 네비게이션 자동 닫힘
   ↓
[카드 리스트 표시]
   ┌──────────────────┐
   │ [본부] 인사발령   │
   │ 부장급 승진...   │
   │ 2025-11-18       │
   │ [☆] [→]          │
   └──────────────────┘
   ↓
[스와이프 제스처]
   - 카드를 왼쪽으로 스와이프
   - 액션 버튼 표시: [북마크] [공유]
   ↓
[북마크 터치]
   - 햅틱 피드백
   - "북마크 추가됨" 알림
   ↓
[공유 터치]
   - 네이티브 공유 시트 표시
   - 카카오톡 선택
   ↓
[메시지 전송]
   - 동료에게 링크 공유
   ↓
[완료]
```

**소요 시간**: 1-2분
**주요 액션**: 필터(1개), 스와이프(1회), 북마크(1건), 공유(1회)

- 세션 만료: "세션이 만료되었습니다" 알림 + 자동 로그아웃

     │    │         │    │
   Yes│    │No     Yes│    │No

  ┌────┐ ┌────┐  ┌────┐ ┌────┐
  │알림│ │롤백│  │알림│ │롤백│
  │표시│ │+   │  │표시│ │+   │

  └────┘ └────┘  └────┘ └────┘

  [완료]  [복구]  [완료]  [복구]
```

**낙관적 업데이트 (Optimistic Update)**:
1. 사용자 클릭 즉시 UI 업데이트 (빠른 피드백)
2. 백그라운드에서 API 호출
3. 성공 시 그대로 유지
4. 실패 시 이전 상태로 롤백 + 에러 알림

**예상 시간**:
- UI 반응: 즉시 (0ms)
- API 왕복: 200-300ms
- 사용자 체감: 즉각적

### 개선 기회 식별

**Funnel 분석**:

```
100% - 홈페이지 방문
 ↓
85% - 전체 게시물 클릭
 ↓
60% - 필터 사용
 ↓
40% - 검색 사용
 ↓
20% - 북마크 추가
 ↓
10% - 데이터 내보내기
```

**Drop-off 포인트 개선**:
1. 홈 → 전체 게시물 (15% 이탈)
   - 개선: 최근 게시물에서 바로 북마크 기능 추가
2. 필터 사용 (25% 감소)
   - 개선: 프리셋 필터 제공 (예: "오늘 입찰공고")
3. 검색 사용 (20% 감소)
   - 개선: 검색 가이드 툴팁, 인기 검색어 추천
